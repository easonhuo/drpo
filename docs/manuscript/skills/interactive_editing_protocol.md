# Interactive Editing Protocol

The interactive editor uses the same skill library as the batch pipeline.  The
protocol below is an execution guide, not a second source of paper facts.

## Section or subsection planning

1. State the unit's unique task.
2. State the reader question.
3. State non-goals.
4. Draw the logic map.
5. Only then draft or revise prose.

## Paragraph blueprint work

1. Verify that the outline is the parent contract.
2. Assign one rhetorical function to each paragraph.
3. Bind each paragraph claim to evidence, allowed conclusions, and forbidden conclusions.
4. Preserve parent order and stable identifiers.
5. Treat child-parent mismatch as a child problem unless the parent has been independently re-triaged.

## Prose generation

1. Generate from the approved blueprint.
2. Keep old information before new information.
3. Use authorized terminology from the relevant project profile.
4. Do not import paper-specific facts from generic exemplars.
5. Do not expand claim scope while polishing language.

## Compression

1. First check that the scientific chain is complete.
2. Remove repetition before removing new information.
3. Move proof detail or exhaustive settings to appendix material.
4. Delete correct but locally non-functional material.
5. Re-run claim-evidence and terminology checks after compression.

## Formula display and numbering

1. Classify each formula as main result, bridge relation, or auxiliary notation.
2. Display and label only formulas that are central or referenced later.
3. Prefer inline notation for local definitions.
4. Check line length and rendered layout.

## Experiment writing

1. Separate mechanism, causal, task-performance, robustness, and implementation claims.
2. Assign each environment or benchmark its evidentiary role.
3. Use terminal evidence for dynamic claims.
4. Keep best-checkpoint and terminal-checkpoint statements separate.
5. Preserve negative, failed, or inconclusive results as constraints on the allowed conclusion.

## Rollback rule for high-risk editing infrastructure

If an editing change would alter default pipeline behavior, generation order,
project-profile precedence, or claim-boundary enforcement, it must have a disable
path and a documented rollback plan before it is enabled by default.
